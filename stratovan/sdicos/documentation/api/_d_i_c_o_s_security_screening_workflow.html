DICOS Usage in the Security Screening Workflow

The following image illustrates how the DICOS file specification fits into the Security Screening Workflow. At points in the Security Workflow where data about an Object Of Inspection needs to be transferred from one device to another DICOS file Specification is used to format the data for sending and the DICOS Networking Protocol is used to send and receive the data.

  1. Objects of Inspection (OOI) such as baggage, cargo, people etc., are passed through security screening devices. These include EDS, DX and AIT devices.

  2. Security devices output DICOS files with the screening data such as CT Volume, time and location of scan, etc. that are sent over a network to other devices for further processing. Security scanning devices may or may not generate a Threat Detection Report (TDR) for the OOI.

  3. The DICOS Transfer protocol is used to transfer these files over a network to other DICOS compatible devices. These follow on devices include Automated Threat Recognition (ATR) devices and On Screen Resolution (OSR) Devices.

  4. Follow on Security Devices can make an Alarm Decision about the object being inspected or forward it on for further processing As an example, an ATR device may not be able to make a definitive decision about an OOI and so it will be forwarded to an OSR device for further inspection by security personnel.

  5. DICOS files can be stored in a database for further processing, future analysis, training etc.
