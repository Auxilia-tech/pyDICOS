Stratovan DICOS Toolkit User Guide

User Guide for Software Developers integrating the Stratovan DICOS implementation.

The Stratovan DICOS Toolkit consists of a software library and tools that implements the NEMA DICOS Standard.

Quick Start Steps

  • The Stratovan DICOS Toolkit is available here https://www.stratovan.com/products/dicos-toolkit.
    • Registration with the Stratovan website is required to download the Toolkit.
    • Supported Platforms:
      • 32 bit / 64 bit Windows VS2017, VS2019 and VS2022
      • 32 bit / 64 bit Linux Fedora 35, Ubuntu 20.04 LTS, Rocky Linux 8, and Centos 7
    • Toolkit Contents
      • bin
        • Conformance executables
        • DICOS File Version Converter executable
      • examples
        • Source code examples for performing various tasks.
        • cmake files for building examples are provided
      • include
        • Header files for Stratovan DICOS APIs
      • lib
        • Static libraries or DLLs
    • Add the DICOS library and header files to Visual Studio solution, or Unix Makefiles etc.
    • Compile examples included with the Toolkit using supplied Visual Studio projects or cmake files.
  • Follow quick start examples below to create a DICOS file for a specific IOD type
    • Quick start example code is included in the DICOS Toolkit distribution.
  • Check the output of the examples using the Conformance Testing Executables
    • Examples write DICOS files to disk. These files can be checked for correctness using the conformance executables.
  • Sample and Exemplar DICOS files used by the examples are located in the Examples/DataFiles folder in the toolkit distribution.

DICOS API Overview

The DICOS Library consists of three distinct APIs:

  • User-LevelAPI
    • The User-Level API consists of objects that abstract away the DICOS Module hierarchy and allow for the creation of DICOS files without a deep understanding of the DICOS standard.
  • Module-LevelAPI
    • The Module-Level API consists of objects and methods that implement DICOS Modules and described in the DICOS specification document. It requires an understanding of the hierarchy of DICOS Modules as presented in the DICOS Specification.
    • In general, one module in the DICOS specification corresponds to a class in the Module-Level API.
  • Tag-Level API
    • The Tag-Level API consists of objects and methods that implement DICOS Attributes and other Module building blocks. It requires a strong understanding of the DICOS specification and requires the user to set attributes using their DICOS Tags.

The majority of examples provided with the Stratovan DICOS Toolkit use the User-Level and Module-Level APIs. It is expected that users will rarely need to use the Tag-Level API.
