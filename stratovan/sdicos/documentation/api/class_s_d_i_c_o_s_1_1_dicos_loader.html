Load DICOS data using a DicosFileListing::OOIFiles object and specify through ID's and UID's the data to load.

Set ID strings to filter the data to load. To ignore an ID for loading, set it to an empty string. Example: Load(vOOIFiles, "1.1.1", "", "3.3.3") will load all files with OOI ID "1.1.1" and series instance UID "3.3.3". Example: Load(vOOIFiles, "", "", "", "4.4.4") will load all files with a SOP class UID "4.4.4".

Parameters
vOOIFilesList of all DICOS files
dsOOIIDObject of inspection ID of data to load. Load all object of inspections by leaving the string empty.
dsScanInstanceUIDScan instance UID of data to load. Load all scans by leaving the string empty.
dsSeriesInstanceUIDSeries instance UID of data to load. Load all series by leaving the string empty.
dsSopClassUIDSOP class UID of data to load. Load all SOP class UID's by leaving the string empty.
dsSopInstanceUIDSOP instance UID of data to load. Load all SOP instances by leaving the string empty.
bFromOpticalDriveWhether or not the file is coming from an optical drive, default to false

void SDICOS::DicosLoader::SetMemoryManager ( IMemoryManager *  pMemMgr)

Provide a memory manager for allocating & deallocating memory for the pixel data.

The DICOS library will internally use the memory manager's allocator for each slice. When the pixel data is to be deleted, the SDK will use the memory manager's deallocator for each slice.
