161 ///
162 /// \class CT
163 /// \brief Represents a single CT scan with original pixel data.
164 ///
165 /// This class abstracts away the hierarchy of the DICOS
166 /// specification and provides an API that contains methods for setting all of
167 /// the attributes needed for a DICOS CT object.
168 ///
169 /// It represents a single Series in a Scan in an Object of Inspection (OOI) with original pixel data.
170 /// Required and optional attributes not explicitly set using the API methods are set to default values.
171 ///
172 /// The volume data for a single CT scan is contained in one or more volumes called Sections
173 /// represented using the SDICOS::Section class.
174 /// Sections are independently allocated.
175 ///
176 /// The two types of Sections are called related and independent.
177 /// Related Sections:
178 /// For CT Data, a larger volume can be broken down into smaller sections along the Z-direction (Depth).
179 /// These sections must shared the same Width(W), Height(H).
180 /// For example a 512x512x512 (WxHxD) volume can be broken down into three Sections
181 /// of dimensions 512x512x100, 512x512x200 and 512x512x212.
182 /// The data can be set for these Sections independently, but when the CT data is written to a file
183 /// the related sections are combined into a single volume and the original Sections are not
184 /// restored when read.
185 ///
186 /// Independent Sections:
187 /// Independent sections cannot be combined to form a larger volume. These are
188 /// Sections that have different dimensions, spacing or location.
189 /// Independent Sections are saved to separate DICOS files.
190 ///
191 /// For example, for a particular OOI and Scan, a Series could have a normal
192 /// resolution CT volume for the whole OOI and a small higher-resolution volume
193 /// for a sub-volume that is of particular interest.
194 /// These are stored as independent sections.
195 ///
