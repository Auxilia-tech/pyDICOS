171  /// (Non-blocking call) Start the server to process DICOS data received by incoming client connections.
172  /// The server will continue to listen until StopListening() is called.
173  /// This method processes incoming connections on a separate thread. Each unique connection starts a DICOS
174  /// session, which is processed in its own thread. Received DICOS data is placed in a processing queue that invokes
175  /// the callback interface. The callback function must exit before the processing queue can send more DICOS data.
176  /// NOTE: Calling StartListening(), StopListening(), StartListening() close enough together can
177  /// trigger 'Address Already in Use' errors.
178  /// @param icallback Callback interface. If the preferred API is set to Module-Level or User-Level API, the original Tag-Level data (excluding pixel data) can be retrieved with DicosData::GetOriginalData() or DicosData::TakeOwnershipOriginal()

180  /// @param nPreferAPI Preferred API for callback

Once the DICOS session ends, the data will be placed into User-Level API objects. For CT and AIT 3D, if data was sent slice by slice, each slice will be placed into a single User-Level API.
